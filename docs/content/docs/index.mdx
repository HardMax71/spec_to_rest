If any check above had returned `unsat`, `compile` would exit non-zero and write nothing &mdash;
not even an empty output directory. See [Verify-as-gate in `compile`](/pipelines/verification#verify-as-gate-in-compile).

## What is a verifying compiler?

Most code generators are scaffolds: they write boilerplate from a schema, then trust you to keep
the runtime correct. A verifying compiler treats the specification as a logical theory. Each
operation produces a verification condition (VC) &mdash; a first-order formula that must be
unsatisfiable for the operation to preserve every invariant. The compiler discharges those VCs
against an SMT solver, and only emits code once every check has been mechanically discharged.
The promise isn't "we generated handlers from your schema." It's "we have a proof that the
generated handlers, on every input the specification admits, leave the invariants intact."
