# UrlShortener

Generated TypeScript service (TypeScript + Express + Prisma + MySQL).

## Quick start

```bash
cp .env.example .env
docker compose up --build
curl http://localhost:8080/health
curl http://localhost:8080/ready
curl http://localhost:8080/metrics
```

Traces are opt-in: set `OTEL_EXPORTER_OTLP_ENDPOINT` (for example
`http://localhost:4318`) to export OTLP spans for every request; leave it
unset and tracing stays off. `OTEL_SERVICE_NAME` overrides the default
service name.

## Local dev

```bash
npm install
npx prisma generate
npm run typecheck
DATABASE_URL='mysql://url_shortener:url_shortener@localhost:3306/url_shortener' npm run dev
```

## Database migrations

Migrations under `prisma/migrations/` are generated by `spec-to-rest compile` — do not
edit `prisma/schema.prisma` or run `prisma migrate dev` directly (it would conflict with
pre-generated migrations).

```bash
make migrate-up                              # npx prisma migrate deploy
make migrate-down NAME=003_schema_update     # run down.sql + DELETE from _prisma_migrations
```

Prisma's CLI has no native `migrate down` command, so `make migrate-down` requires the
exact migration folder name (rather than an implicit "latest" — which would be unreliable
when there are pending un-applied migrations on disk). Run `make migrate-down` without
`NAME=` to see the list of available migration folders.

To change the schema, edit the upstream `.spec` and re-run `spec-to-rest compile`; the
compiler emits a new `NNN_schema_update/` directory with both `migration.sql` and
`down.sql` derived from the spec diff.
